Red Flag Features

If an app has any of these features, it exceeds the 8-week timeline. Auto-reject or extend timeline significantly.

🔄

Real-Time Multiplayer

WebSocket infrastructure, state sync, conflict resolution. Adds +6-8 weeks.

📝

UGC with Moderation

Content review, abuse prevention, legal liability. Adds +4-6 weeks.

🎬

Video/Photo Editing

Media processing, filters, rendering. Adds +8-12 weeks.

📴

Offline-First Architecture

Local database, sync engine, conflict resolution. Adds +3-4 weeks.

👥

Social Networking

Profiles, feeds, following, activity streams. Adds +6-8 weeks.

🛒

Complex E-commerce

Multi-vendor, inventory, orders, disputes. Adds +8-12 weeks.

🗺

Custom Maps/Geolocation

Routing, geofencing, real-time tracking. Adds +4-6 weeks.

🏥

HIPAA/PCI Compliance

Security audits, legal review, testing. Adds +3-6 weeks.

Scope Boundaries

What's in scope for an 8-week rebuild vs what should be deferred or rejected.

In Scope (8 Weeks)

  • Simple CRUD apps (5-10 screens)
  • 1-3 core features only
  • BaaS backend (Firebase/Supabase)
  • Basic OAuth authentication
  • Single platform (iOS OR Android)
  • Standard UI patterns
  • Basic analytics integration
  • Crash reporting
  • Simple subscription/IAP

Out of Scope (Defer or Reject)

  • Real-time collaboration features
  • Video/audio processing
  • User-generated content moderation
  • Social networking features
  • Complex e-commerce/marketplace
  • Offline-first with sync
  • Custom mapping/routing
  • AI/ML custom models
  • Both platforms simultaneously

App Type Guide

Quick reference for which app types to accept, approach with caution, or avoid.

Accept (Tier 1)

  • Habit trackers (6-7 wk)
  • Expense trackers (7-8 wk)
  • Focus/productivity timers (5-6 wk)
  • Flashcard apps (6-8 wk)
  • Simple utilities (4-6 wk)
  • Educational quizzes (6-8 wk)
  • Note-taking (text only)
  • Simple calculators

Caution (Tier 2)

  • Fitness trackers (static library only)
  • Recipe apps (provided database)
  • Journal apps (text only, no media)
  • Budget apps (manual entry only)
  • Weather apps (API integration)
  • Simple CRM tools
  • Booking/scheduling
  • Inventory trackers

Avoid (Tier 3)

  • Social networks
  • Marketplace apps
  • Media editors (photo/video)
  • Real-time collaboration tools
  • Dating apps
  • Streaming platforms
  • Healthcare/fintech (compliance)
  • Multi-tenant B2B SaaS

Assessment Checklist

Use this checklist during initial app evaluation before committing to a partnership.

Technical Due Diligence

  • Reviewed app in App Store / Play Store
  • Counted screens (target: <10)
  • Identified core features (target: 1-3)
  • Checked for red flag features
  • Assessed backend complexity
  • Checked for compliance requirements
  • Evaluated data migration needs
  • Calculated complexity score

Developer Discussion Points

  • Understand current tech stack
  • Access to existing codebase (read-only)
  • Analytics access (identify high-engagement features)
  • User data export requirements
  • Feature priorities (what can be cut?)
  • Platform priorities (iOS first? Android first?)

Go/No-Go Decision

  • Complexity score is 16 or below (Tier 1)
  • No red flag features present
  • Developer agrees to scope constraints
  • Monetization opportunity is clear
  • Timeline expectations are aligned

Real-World Evidence

Case studies and industry data supporting timeline estimates.

Dashlane iOS Rewrite

Timeline: 6 months with 4 dedicated developers + 5 maintaining legacy app.
App Type: Password manager (complex security app).
Approach: Modernization (not complete rebuild) - reused Swift crypto/sync frameworks, new SwiftUI + UIKit hybrid UI.
Trade-offs: Features frozen during rewrite, some cut from initial launch.
Results: 50% fewer crashes, 50% faster account loading.
Key Lesson: Even a well-resourced team needed 6 months for a complex app. Simple apps can be faster.

Source Finding
Multiple Agencies Simple apps: 8-12 weeks | Medium: 16-24 weeks | Complex: 24-48+ weeks
Panthos Labs MVP development possible in 4 weeks with strict scope control
Scorchsoft Velocity Sprints: 2-8 weeks for focused MVPs
Industry Consensus Rewrites with existing specs can be 20-40% faster than new builds
Cross-Platform Data Flutter/React Native: 30-50% faster than dual native development
Team Size Impact Solo developer: 2-3x slower than 2-3 person team

Implications for Moonshot

What this research means for the partnership model.

Target Tier 1 Apps

Focus on habit trackers, expense apps, timers, flashcards, and simple utilities. These fit reliably in 8 weeks and have proven monetization potential.

Use the Scorecard

Score every potential partnership before committing. Apps scoring 17+ should be approached with caution or declined to protect timeline reliability.

Mandate BaaS Backends

Firebase or Supabase saves 3-4 weeks vs custom backend. Only accept custom backends if they're well-documented and genuinely reusable.

Single Platform First

Building both iOS and Android doubles timeline. Start with the platform where 80%+ of users are, add the other in Phase 2.

Native Rewrite Philosophy

Why native development may be preferred for long-term portfolio apps.

Why Native?

  • Performance - Better UX, faster load times, smoother animations
  • Monetization - Native ad SDKs perform better
  • Analytics - Deeper integration with platform analytics
  • Maintenance - Single codebase per platform, no framework updates

Tradeoffs

  • Higher initial cost (two codebases)
  • Longer timeline vs cross-platform
  • Need expertise in both iOS and Android

Android Stack

  • Language: Kotlin
  • Architecture: MVVM + Clean Architecture
  • UI: Jetpack Compose (or XML)
  • DI: Hilt
  • Networking: Retrofit + OkHttp
  • Local storage: Room
  • Analytics: Firebase Analytics, Mixpanel
  • Ads: AdMob, ironSource, Unity Ads
  • Crash reporting: Firebase Crashlytics

iOS Stack

  • Language: Swift
  • Architecture: MVVM + Combine
  • UI: SwiftUI (or UIKit)
  • Networking: URLSession / Alamofire
  • Local storage: Core Data / SwiftData
  • Analytics: Firebase Analytics, Mixpanel
  • Ads: AdMob, ironSource
  • Crash reporting: Firebase Crashlytics
